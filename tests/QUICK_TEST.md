# Быстрая проверка парсера

## Шаг 1: Проверка логов парсера

Парсер теперь логирует информацию в error_log. Проверьте логи:

```bash
# Просмотр последних логов PHP
tail -f /var/log/php/error.log
# или
tail -f /var/www/kp/api/error_log
```

## Шаг 2: Тест через браузер

1. Откройте `torgikp.html` в браузере
2. Откройте консоль разработчика (F12)
3. Вставьте ссылку на торги:
   ```
   https://www.b2b-center.ru/market/priobretenie-reduktora-konveiera-ch-5a028-0-sztm-dlia-ao-mikhailovskii/tender-4242870/
   ```
4. Нажмите "Загрузить данные из ссылки"
5. Проверьте консоль - должны быть логи:
   - `Запрос к API: ...`
   - `Ответ от API: ...`
   - `Найдено позиций: ...`
   - `Применяем обновления: ...`

## Шаг 3: Проверка результата

После загрузки данных проверьте:
- ✅ Поле "Номер торгов" заполнено: `4242870`
- ✅ Поле "Торговая площадка" выбрано: `B2B-Center`
- ✅ **Поле "Описание товара" в таблице заполнено**: `Редуктор 5А028-0 СЗТМ`
- ✅ Поле "Количество" заполнено: `1`

## Если данные не загружаются

### Проблема: 401 Unauthorized
- Убедитесь, что вы авторизованы на сайте
- Проверьте настройки авторизации на веб-сервере

### Проблема: Данные не заполняются в форму
1. Проверьте консоль браузера на наличие ошибок
2. Проверьте ответ API в Network tab:
   - Откройте DevTools → Network
   - Найдите запрос к `parse-tender-data.php`
   - Проверьте Response - должен быть JSON с полем `items`

### Проблема: Парсер не находит данные
1. Проверьте логи PHP (см. Шаг 1)
2. Убедитесь, что страница B2B-Center доступна без авторизации
3. Возможно, структура HTML изменилась - проверьте селектор `tr[class="c2"]`

## Отладка

Если проблема сохраняется, соберите следующую информацию:

1. **Ответ API** (из Network tab):
   ```json
   {
     "success": true/false,
     "data": {...},
     "platform": "b2b-center"
   }
   ```

2. **Логи из консоли браузера** (все console.log)

3. **Логи PHP** (error_log)

4. **HTTP код ответа** от API

Отправьте эту информацию для дальнейшей диагностики.
